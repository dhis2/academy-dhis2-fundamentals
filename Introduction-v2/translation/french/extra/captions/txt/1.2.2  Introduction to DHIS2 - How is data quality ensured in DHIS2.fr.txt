- Examiner les données dans DHIS2 / Introduction à DHIS2

- [Instructeur] Une fois les données saisies dans la plateforme, comment pouvons-nous vérifier leur exactitude et leur fiabilité ? Dans cette vidéo, nous allons explorer plusieurs outils de l'application Qualité de données de DHIS2 qui nous aideront à examiner et à améliorer la qualité de nos données. Tout d'abord, dans l'application Qualité de données, l'outil Analyse de règles de validation vous permet de tester les règles de validation par rapport aux données enregistrées dans le système. Après avoir lancé la vérification, vous obtiendrez un rapport contenant une liste des données qui doivent être vérifiées. Par exemple, si nous regardons la première ligne, notre règle de validation dit que la valeur enregistrée pour la 3ème consultation prénatale, ou CPN ne peut pas être supérieure à la valeur enregistrée pour la 2ème consultation prénatale. Mais la valeur enregistrée pour la CPN3 est de 6, alors que la valeur de la CPN2 est de 4. Nous pouvons faire un suivi auprès de cet établissement pour déterminer la cause de l'erreur. Notez que vous pouvez également effectuer l'analyse des règles de validation pendant le processus de saisie des données. La Détection des valeurs atypiques, basée sur l'écart-type est un autre outil que nous pouvons utiliser pour vérifier la qualité des données, en identifiant les valeurs qui sont potentiellement aberrantes par rapport à la loi normale standard des données examinées. Comme pour l'analyse des règles de validation, lorsque nous lançons une analyse des valeurs aberrantes, basée sur l'écart-type, nous obtenons un rapport contenant toutes les données qui doivent être vérifiées. Par exemple, sur cette ligne, les données indiquent que seuls 5 vaccins BCG ont été administrés dans cet établissement de santé. Cette valeur de 5 se situe en dehors de la loi normale appliquée à ces données, et elle a été identifiée comme une source potentielle d'erreur. Un autre outil est la Détection des valeurs atypiques, cette fois-ci basée sur les minimum et maximum, qui vérifie si les données se situent dans une fourchette spécifiquement prédéfinie. Cet outil répertorie les valeurs qui se situent en dehors de la fourchette prédéfinie. Nous pouvons voir dans l'exemple, une valeur de 45 cas suspects de MAPI, ce qui est en dehors de la fourchette avec un minimum de 0 et un maximum de 22. Le 4ème et dernier outil disponible dans l'application Qualité de données est l'outil Analyse des Suivis. Analyse des Suivis crée une liste de toutes les valeurs de données marquées pour le suivi. Nous pouvons marquer la valeur d'une donnée pour un suivi dans l'application Saisie de données ou dans l'application Qualité de données via les rapports générés par les outils d'analyse des valeurs aberrantes utilisant l'écart-type et les valeurs minimum et maximum. Voici un exemple de rapport obtenu après avoir exécuté une analyse de suivi dans DHIS2. En coordination avec l'OMS, un outil de qualité des données pour DHIS2 a été créé. Cette application génère des résultats sur la qualité des données en suivant le cadre d'examen de la qualité des données de l'OMS. Celui-ci inclut l'exhaustivité : est-ce que toutes les données attendues sont enregistrées ; la ponctualité : est-ce que les données ont été reçues à temps ; la cohérence interne, qui compare les données soumises en interne entre elles, et la cohérence externe, qui compare les données avec d'autres sources, comme les enquêtes. Examinons par exemple la cohérence interne. L'outil de qualité des données de l'OMS nous permet d'identifier les valeurs aberrantes dans nos données. Nous pouvons en voir un exemple dans le district C-1, où le nombre de vaccins contre la rougeole administrés en janvier 2020 est beaucoup plus élevé que les autres valeurs signalées dans le même district la même année. Cela nous aide à visualiser facilement les données qui doivent être vérifiées avant de procéder à une analyse. En résumé, dans DHIS2, il existe plusieurs outils qui nous aident à vérifier la qualité des données saisies, comme les règles de validation, l'analyse des valeurs aberrantes basée l'écart-type l'analyse des valeurs aberrantes basée sur les minimum et maximum et l'analyse de suivi. Le DHIS2 et l'OMS ont également collaboré pour créer un outil de qualité des données qui effectue des contrôles pour valider l'exhaustivité, la ponctualité la cohérence interne et la cohérence externe.
