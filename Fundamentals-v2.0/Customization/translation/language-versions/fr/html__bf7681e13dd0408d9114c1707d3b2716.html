Résumé du module 2

Vous avez maintenant atteint la fin du deuxième module du cours de personnalisation du système DHIS2. Dans ce module, vous avez appris:

  • Comment créer une hiérarchie d'unités d'organisation dans l'application Maintenance, y compris:
    • Infrastructures
    • Districts
    • Régions
  • Que la collecte des données se fait souvent au niveau des établissements (par exemple, les centres de santé, les hôpitaux, les écoles) et qu'elles sont ensuite agrégées vers les niveaux supérieurs de la hiérarchie.
  • La forme de la hiérarchie et le nombre de niveaux dépendent du contexte de chaque mise en œuvre.
  • Pour créer des unités d'organisation:
    • Ouvrez l'application Maintenance
    • Cliquez sur l'onglet "Unité d'organisation" en haut de la page, puis sélectionnez "Unité d'organisation" dans le menu de gauche pour accéder à la page de gestion des unités d'organisation.
    • Dans la hiérarchie des unités d'organisation, sélectionnez l'unité d'organisation mère.
    • Cliquez sur le signe plus (+) bleu dans le coin inférieur droit de l'écran.
    • Remplissez les champs nom, nom court, code et date d'ouverture.
      • La date d'ouverture est un champ obligatoire
      • Vous ne pouvez pas introduire de données dans l'installation avant la date d'ouverture.
    • Cliquez sur "Enregistrer".
  • Les groupes d'unités d'organisation sont utilisés pour créer une hiérarchie organisationnelle alternative, généralement appliquée au niveau des installations.
  • Les unités d'organisation peuvent être regroupées, par exemple, par:
    • Type de services fournis par l'établissement
    • Si l'établissemet fonctionne dans le contexte rural ou urbain
    • Type de propriété de l'établissement
  • Pour créer des groupes d'unités d'organisation:
    • Ouvrez l'application Maintenance
    • Cliquez sur l'onglet "Unité d'organisation" en haut de la page.
    • Sélectionnez le groupe d'unités d'organisation dans le menu de gauche.
    • Sélectionnez le signe plus (+) bleu pour ajouter un groupe d'unités d'organisation.
    • Remplissez les champs nom, nom court et code
    • Sélectionnez un symbole pour représenter le groupe dans les applications analytiques.
    • Sélectionnez les unités d'organisation à inclure dans le groupe d'unités d'organisation.
  • Les groupes d'unités d'organisation peuvent à leur tour être organisés en ensembles de groupes.
  • Les ensembles de groupes peuvent apporter une dimensionnalité supplémentaire à l'analyse des données, car les données sont facilement filtrées, organisées ou agrégées par groupes au sein d'un ensemble de groupes.
  • Si l'ensemble de groupes d'unités d'organisation est utilisé comme dimension de données, chaque unité d'organisation ne doit être membre que d'un seul groupe au sein de l'ensemble de groupes. Cela permet d'éviter un double comptage des données si la dimension est utilisée dans l'analyse.
  • Pour créer des ensembles de groupes d'unités d'organisation:
    • Ouvrez l'application Maintenance
    • Cliquez sur l'onglet "Unité d'organisation" en haut de la page.
    • Sélectionnez l'ensemble des groupes d'unités d'organisation dans le menu de gauche.
    • Sélectionnez le signe plus (+) bleu pour ajouter un ensemble de groupes d'unités d'organisation.
    • Remplissez les champs nom, nom court et code
    • Sélectionnez les groupes d'unités d'organisation à inclure dans l'ensemble de groupes d'unités d'organisation.
