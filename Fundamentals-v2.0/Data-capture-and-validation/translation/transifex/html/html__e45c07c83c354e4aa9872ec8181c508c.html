Overview

In this activity, you will run an Outlier Detection to check for any outliers in a data set.

Instructions

Image 1 - An Outlier Detection table with identified outliers

Run Outlier Detection:
  1. Sign in to DHIS2 Trainingland (Validation)
  2. Open the Data Quality app from the Apps Menu
  3. Select the Outlier Detection tool
  4. Select your Outlier Detection Dimensions and other options:
    • Select COVID-19 Surveillance (Aggregate) Data Set from the dropdown menu
    • Select the Cat District as the Organisation Unit
    • Select the Period: July 1, 2021-September 30, 2021
    • Select the Outlier Detection Method: z-score
    • Select the threshold of 3 (this sets how many standard deviations the data are allowed to deviate from the mean before they are classified as an outlier; 3 is the standard)
    • Select the maximum number of results you want to view
  5. Select ‘Start’
  6. Analyze the results of the Outlier Detection Report
    • What organisation units have outliers?
    • Look at the various values, z-scores and standard deviations
    • Are there any values marked for follow-up?
  7. When you are finished analyzing the results, run another Outlier Detection Report with the same dimensions, except this time change the threshold to 4. How does this impact the results?
Explore DHIS2

If you’d like additional practice in DHIS2, try these optional, exploratory steps below:

  1. In the Data Quality app, use the min/max values Outlier Detection Method for 2 or more facility level organisation units in Animal Region. How does the data value compare to the min/max limits? Which data elements would you mark for follow-up?
  2. Use the z-score Outlier Detection Method to look for outliers in the Food Region or facility level organisation units. Try setting a threshold lower than the standard to see how it impacts the results
