Reports app

The Reports app is a great tool within DHIS2 for viewing data entered for specific data dimensions. You can also look at data quality aspects of your data to check the data are complete and have been entered in a timely fashion.

Once data has been entered in DHIS2, the Reports app can give you a more holistic view of the data by data set. This is useful for program managers to view all data entered for specific organisation units, periods and data sets. The Reports app contains numerous different reports, such as the Standard Report, Data Set Report, Reporting Rate Summary, Resource and Organisation Unit Distribution Report. In this module we’ll be focusing on two of these reports: the Data Set Report and the Reporting Rate Summary.

The Data Set Report allows you to view your aggregated data by data set. The data in this report are aggregated by the Organisation Unit Hierarchy. For example, you can select a district and period within a data set, and the report will give you all the values from any facilities in the selected time period below that in the Data Set Report.

The Reporting Rate Summary shows you how many data sets have been submitted by organisation unit and period, allowing you to see how often data are updated and entered into your system. The Reporting Rate Summary is based on completed data sets (those that have the complete button pressed in the Data Entry app). There are five different parameters available from the Reporting Rate Summary:

  • Actual Reports, which are how many have been received
  • Expected Reports, which are how many are expected to have been submitted
  • Reporting Rates, which is the percentage of reports registered as complete based on the number expected
  • Actual Reports on Time and Reporting Rate on Time, which each present the data based on a timeliness factor on when data sets were completed
