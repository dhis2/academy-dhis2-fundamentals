Import/Export app: Importing data

Image 1 - Importing data using the Import/Export app

What if you have a lot of historical data or a spreadsheet of data that needs to be added to DHIS2? Do you have to enter these data manually in DHIS2? The Import/Export app is another way to get data into DHIS2 without entering it by hand. The Import/Export app is used for importing metadata and data into DHIS2 and exporting metadata and data from DHIS2. However, this section will only focus on importing data.

To import data in the Import/Export app, you must select the Data Import tab. The app can import data values from multiple file formats including PDF, JSON, CSV, ADX XML and DXF 2 XML. You can select the file format and upload the file.

The app includes four options or strategies for importing data:

  1. Merge: Import new values and update existing values

  2. Append: Import new values only

  3. Update: Only update existing values, ignore new values

  4. Delete: Remove values included in the uploaded file

In addition, for large data imports, you can select the option to preheat the cache to make the large import process faster. There is also an option to skip the audit, meaning audit values will not be generated, as well as advanced importing options.

The app allows you to launch a “dry run” to test the import settings without importing any data. It is highly recommended to use the dry run option to test the import and check for problems. The app will show an import job summary with the results of the dry run, and any conflicts or errors displayed in the table under the import summary. After a successful dry run, indicated by a green completed label, data can be imported into the system.

For more information on importing data, review the DHIS2 Import/Export app documentation.
