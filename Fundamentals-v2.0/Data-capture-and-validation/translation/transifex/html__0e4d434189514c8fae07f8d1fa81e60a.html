Overview

In this activity, you will run a Batch Validation Rule Analysis using the Data Quality app.

Instructions

Image 1: The results of a Batch Validation Rule Analysis report, showing which validation rules have been broken

Run a Batch Validation Rule Analysis:
  1. Sign in to DHIS2 Trainingland (Validation)
  2. Open the Data Quality app from the Apps Menu
  3. Select Validation Rule Analysis
  4. Select your Validation Rule Analysis Dimensions:
    • Select Animal Region in the organisation unit hierarchy
    • Select July 1, 2021 as the start date and December 31, 2021 as the end date
    • Select COVID-19 as the validation rule group
  5. Select ‘Validate’. Note the results may take a few seconds to generate
  6. Analyze the result of the Validation Rule Analysis
    • What kind of validation rules are broken?
    • What is the level of importance for the validation rule?
    • Are there validation rules that are repeatedly broken?
  7. Select the information icon to find out more information about one of the validation rules
    • What is the left side and right side of the validation rule expression?
    • What would the data value need to be for the validation rule to pass?
