Module Summary

You have now reached the end of the third module for the Data Capture and Validation course. In this module, you have learned:

  • Data quality:
    • Data elements have different value types
      • While primarily numerical, data elements can also be text, check boxes or date ranges, for example
      • Entering an invalid value type will cause a data element to turn yellow and you will receive a notification at the top of your web browser informing you that your value type is incorrect
    • The Run Validation button performs a validation check on data entered into the Data Entry app
      • Clicking the Run Validation button will run a validation rules check, configured for the elements within the data set
      • Any errors will be identified within the data set
    • The Complete button submits a data set as complete to the DHIS2 server
      • Select Complete at the bottom of the Data Entry form when you have finished entering data into a data set
      • You will be notified of any validation errors in the data set
      • You can still edit the data after it has been marked as complete
  • Batch Validation Rule Analysis:
    • Validation can be run on aggregated organisation units
      • Select the Validation Rule Analysis from the Data Quality app
      • Set the organisation unit, period and validation rule to check, then select ‘Validate’
      • Check any errors that appear
  • Outlier Detection:
    • Outlier Detection finds outliers from aggregated data in DHIS2
      • Select the Outlier Detection from the Data Quality app
      • Select the data set, organisation unit and time period you want to check for outliers
      • Select the detection method and maximum number of outliers then select start
      • The resulting table will show any outliers detected, based on the detection method you have chosen
  • Follow-Up Analysis:
    • Follow-Up Analysis can check to see any data in your selected data set has been marked for follow-up
      • Open the DHIS2 Data Quality app and select Follow-up Analysis
      • Select the data set, organisation unit and time period you want to check for follow-up
      • Click the Follow-Up button
      • Review the chart showing the data selected for follow-up
  • Reports app:
    • Reports can be generated about data entered in the Data Entry app
    • The Data Set Report can generate reports about specific data sets
      • Open the Reports app and select the Data Set Report
      • Select the data set and other dimensions you want to run a report for
      • Select if you want to just view the data of the specific organisation unit and not all the organisation units that report into it
      • Click ‘Get Report’
      • You will then be able to review the Data Set Report
    • The Reporting Rate Summary keeps a track of data based on when the complete button submitted the data set as complete to the server
      • Open the Reports app and select the Reporting Rate Summary
      • Select the dimensions you want to run a report for
      • Select if you want to just view the data of the specific organisation unit and not all the organisation units that report into it
      • Click ‘Get Report’
      • You will then be able to review the Actual Reports, Expected Reports, Reporting Rate, Actual Reports on Time and Reporting Rate on Time that appear
