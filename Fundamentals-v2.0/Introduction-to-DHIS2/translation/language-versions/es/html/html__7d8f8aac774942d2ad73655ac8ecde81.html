¿Qué es la interoperabilidad?

En un sistema nacional de salud, es común tener varias plataformas de gestión de información diferentes que realizan funciones complementarias entre sí. Estos sistemas podrían necesitan intercambiar información - o "conversar" entre sí - para que quienes toman las decisiones puedan obtener una visión más amplia de lo que está pasando en el sistema de salud como un todo. La capacidad para enviar y recibir información entre diferentes sistemas, softwares y formatos de archivos se conoce como interoperabilidad.

Por ejemplo, DHIS2 y OpenMRS (Medical Records System) son interoperables, ya que permiten compartir datos y definiciones de datos entre sí. En este ejemplo, un sistema de salud podría usar OpenMRS para almacenar el historial médico de cada admisión hospitalaria individual (entre otros datos hospitalarios). Al mismo tiempo, la implementación de DHIS2 del sistema de salud podría solamente requerir el número de datos agregados de las admisiones hospitalarias separadas por varios criterios. La interoperabilidad permite que DHIS2 reciba estos totales desde OpenMRS y los use para analizarlos en combinación con otros datos almacenados en DHIS2. Esto se conoce comúnmente como integración 1:1, ya que los datos de un sistema se vinculan a otro de manera directa.

¿Cómo funciona la interoperabilidad?

La interoperabilidad depende de estándares para formatos de datos, interfaces, códigos y terminologías. En una situación ideal, estos se obtendrían de estándares internacionales. Sin embargo, en la mayoría de los contextos, se usa una mezcla de estándares internacionales y locales. Estos estándares se pueden dividir en dos niveles:

  1. Contenido: Aseguran que lo que se comparte entre los sistemas representan conceptualmente lo mismo.
  2. Técnico: Tiene que ver con el mecanismo usado para transmitir los datos de un sistema al otro.

A nivel del contenido , se necesita consistencia en las definiciones. Esto incluye no solo consistencia en las definiciones de los elementos de datos, sino también en sus desagregaciones. Volviendo a nuestro ejemplo de OpenMRS, si el elemento de datos "admisiones hospitalarias" en DHIS2 no incluye las desagregaciones requeridas desde OpenMRS (tales como edad y sexo) debido a que las admisiones se han definido como valores totales, implicaría dificultades a la hora de compartir estos datos.

A nivel técnico, se pueden adoptar varios estándares de intercambio de datos. Un ejemplo de estándar para datos agregados se denomina ADX (Aggregate Data eXchange), el cual se usa para compartir datos agregados a través de varios sistemas, incluyendo DHIS2. Otro estándar usado de manera común es FHIR (Fast Healthcare Interoperability Resource), que sirve para intercambiar información de salud de manera electrónica. Tanto HISP como al comunidad de DHIS2 están activamente involucrados apoyando el desarrollo y uso de estándares globales como estos.

Interoperabilidad entre DHIS2 y otros sistemas

Uno de los escenarios de interoperabilidad de DHIS2 más frecuente de encontrar en el uso de DHIS2 como almacén de datos. Este término describe cómo DHIS2 puede usarse para recibir información de múltiples sistemas y fuentes para facilitar la triangulación de datos. En este escenario, DHIS2 actúa usualmente como estándar para los metadatos compartidos por esos sistemas conectados, permitiendo la alineación de códigos, definiciones e identificadores para la información intercambiada.

Vamos a ver un ejemplo de cómo un sistema específico y compatible con ADX puede vincularse con DHIS2:

  • iHRIS: Sistema para rastrear y gestionar datos trabajadores de la salud.
  • OpenMRS: Sistema de registros médicos
  • OpenELIS: Sistema de información de empresas de laboratorios

Como muestra el diagrama, iHRIS puede enviar datos relevantes a DHIS2, como el "número de médicos", "número de enfermeras" y "número total de empleados". OpenMRS es capaz de agregar los datos de enfermedades de los pacientes y enviar esos datos agregados a DHIS2. OpenELIS puede generar datos sobre los resultados de los tests de laboratorio y enviarlos a DHIS2. En esta red interoperable, la aplicación de DHIS2 sirve como almacén de datos integrado que aloja datos desde varias fuentes, donde los datos pueden usarse a nivel nacional para tomar decisiones informadas.

Existen escenarios adicionales de interoperabilidad donde DHIS2 puede jugar un papel más pequeño. En estos escenarios, en vez de usar DHIS2 para combinar datos desde otros sistemas, pueden enviarse datos desde DHIS2 a otros sistemas que los requieran. Esto puede incluir sistemas nacionales, globales o regionales que necesiten datos de DHIS2. Por ejemplo, DHIS2 puede enviar datos nacionales de vigilancia de enfermedades a sistemas de información de vigilancia regionales para facilitar la vigilancia fronteriza o regional.

Mientras que estos modelos de interoperabilidad e intercambio de datos son realizables, en la práctica muchas veces suponen desafíos a superar. Muchos de los sistemas involucrados en el intercambio de información afrontarán desarrollos constantes y dinámicos a través del tiempo; esto puede incluir cambios en el contenido como también en las revisiones de las versiones del software o actualizaciones técnicas. Como resultado, un intercambio que funciona hasta un cierto punto podría no funcionar en el futuro debido a los desarrollos constantes de los sistemas. La interoperabilidad es un proceso que requiere una gobernanza sólida para coordinar el contenido a través de los sistemas, así como habilidades técnicas disponibles de manera constante tanto para la implementación como la modificación de los sistemas.
